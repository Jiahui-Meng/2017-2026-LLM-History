# 架构对比：官方结构字段矩阵

本页只记录官方披露的结构事实。闭源模型不填参数、层数、attention 或专家数量；开放权重模型也只填官方 config/model card 明确给出的字段。工具调用、搜索、computer use、MCP、subagents、Codex/Claude Code 等放在“推理/产品系统结构”，不写成 base model 内部结构。

## 先看结构演进主线

1. `早期主流是 dense Transformer。` 从 Transformer 到 GPT-3，核心矛盾是如何扩大容量，同时保持训练可行。[S036][S037][S040]
2. `2023 以后 MoE 进入主流。` Mixtral 先把 sparse MoE 打成开源标配，后续 DeepSeek、Qwen、Kimi、MiniMax、GLM、Llama 4、Mistral Large 3 全部沿不同方式跟进。[S056][S028][S030][S031][S029][S032][S033][S035]
3. `attention 不再只有一种写法。` Qwen 公开 hybrid attention；DeepSeek/GLM 公开 MLA/DSA 风格字段；MiniMax 公开 qk_norm 与 KV 头配置。[S028][S029][S030][S031][S032]
4. `多模态已进入结构层。` Kimi、Qwen、Llama 4 明确把 vision 结构或 image-text-to-text 路线写进公开配置/模型卡；闭源家族多只公开能力层，不公开内部编码器。[S027][S030][S031][S033][S034]

## 结构差异要点

1. **闭源前沿模型的结构信息主要停留在接口层。** OpenAI、Claude、Gemini、Grok 公开 context、reasoning effort/thinking、工具和多模态输入，但参数、层数、hidden size、attention 与 tokenizer 均官方未披露。[S001: release][S005: release][S008: models][S024: API table]
2. **开放权重 MoE 的披露粒度差异很大。** Kimi、DeepSeek、Qwen、MiniMax、GLM 提供 config 级字段；Mistral 和 Llama 公开总/激活规模和 MoE，但未给出完整层级结构。[S027: config.json][S028: config.json][S030: config.json][S033: model card][S035: model card]
3. **attention 机制不是统一趋势。** Qwen 公开 alternating linear/full attention；DeepSeek 和 GLM 公开 DSA/index 与 LoRA-rank 风格字段；MiniMax 公开 qk_norm 与 RoPE；闭源模型均未披露内部 attention。[S030: config.json][S028: config.json][S032: config.json][S029: config.json]
4. **多模态结构可审计程度不一。** Qwen 和 Kimi 有 vision_config 字段；Gemini、GPT、Claude、Llama、Mistral 公开多模态能力但没有完整 vision encoder 配置。[S027: config.json][S030: config.json][S008: model table][S033: model card]

## 与 LMArena 的边界

LMArena 是人类偏好评测，字段包括 rank、rating、rating_lower、rating_upper、vote_count、category、model_name、leaderboard_publish_date。它可用于观察用户偏好和置信区间，但不能填补未披露的结构字段，也不能证明某个 MoE/attention 设计直接导致榜单分数。[S050: schema]
