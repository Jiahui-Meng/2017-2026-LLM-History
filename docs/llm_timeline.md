# LLM 全局时间线

本页按“能力跃迁”而不是“厂商分组”来理解 LLM 发展史。每个节点都回答 4 个问题：它比前代强在哪里、核心创新是什么、为什么重要、还没解决什么。

## 1. Transformer 基础期 (2017-2019)

### Transformer (2017)

- 比前代强在哪里：训练并行性更高，更能处理长距离依赖，明显缓解 RNN/LSTM 的串行瓶颈。[S036]
- 核心创新：self-attention 成为统一序列建模核心，取代“循环”作为主干计算图。[S036]
- 为什么重要：今天几乎所有主流 LLM 家族都建立在 Transformer 及其变体上。[S036]
- 局限性：当时还没有证明仅靠扩大语言模型就能获得强 few-shot 推理能力。

### GPT-1 (2018)

- 比前代强在哪里：把无监督预训练迁移到下游任务的收益系统化。[S037]
- 核心创新：生成式预训练，然后再用少量监督微调。[S037]
- 为什么重要：GPT 家族从这里确立了“一个大语言模型适配多个任务”的路线。[S037]
- 局限性：规模还不够大，通用泛化能力远未达到后来的 GPT-3 水平。

### BERT (2018)

- 比前代强在哪里：在理解类任务上大幅超过前代单向语言模型。[S038]
- 核心创新：双向表示学习与 masked language modeling。[S038]
- 为什么重要：证明 encoder 路线在检索、分类、问答中极具价值，并长期影响 RAG、embedding 和 reranker 体系。[S038]
- 局限性：不是生成式主干，无法直接承担后来的通用对话式生成任务。

### T5 (2019)

- 比前代强在哪里：把不同 NLP 任务统一成一个输入输出接口，迁移更干净。[S039]
- 核心创新：text-to-text 统一范式。[S039]
- 为什么重要：后来大量 instruction tuning、API 设计和提示模板都受这一路线影响。[S039]
- 局限性：当时行业还没把“对齐”和“产品交互”放到和预训练同等重要的位置。

## 2. 规模涌现期 (2020-2021)

### GPT-3 (2020)

- 比前代强在哪里：few-shot 与 zero-shot 能力显著增强，开始出现明显的 in-context learning。[S040]
- 核心创新：大规模参数、数据和算力扩展带来的涌现式能力。[S040]
- 为什么重要：行业由此普遍接受“规模本身就是方法”的观点。[S040]
- 局限性：会说不等于会按人类意图做事，幻觉、越狱和对话稳定性都很差。

### FLAN (2021)

- 比前代强在哪里：同类底座在看过指令化数据后，更能执行用户具体任务。[S042]
- 核心创新：instruction tuning 被明确证明可显著提升 zero-shot 任务泛化。[S042]
- 为什么重要：后来的 chat、assistant、tool-call 都离不开 instruction-following 基础。[S042]
- 局限性：仍不足以解决长期对齐、拒答和复杂多轮交互问题。

## 3. 对齐与聊天期 (2022-2023)

### Chinchilla (2022)

- 比前代强在哪里：在固定算力预算下给出更优训练配比，效率高于“只堆参数”。[S041]
- 核心创新：compute-optimal scaling law。[S041]
- 为什么重要：改变了后续基础模型的训练经济学，推动行业重视高质量 token 和合理参数/token 比例。[S041]
- 局限性：它优化的是预训练效率，不直接解决用户交互和安全对齐。

### InstructGPT / ChatGPT (2022)

- 比前代强在哪里：回答更贴近人类意图，多轮对话更稳定，也更愿意拒答不该做的任务。[S043]
- 核心创新：RLHF 与聊天产品化。[S043]
- 为什么重要：这是 LLM 从研究模型走向大众入口的拐点。[S043]
- 局限性：产品层成功掩盖不了 base model 在事实可靠性和复杂推理上的不足。

### Claude 1 / Claude 2 (2023)

- 比前代强在哪里：对安全、可控和长上下文的强调更强。[S045][S046]
- 核心创新：Constitutional AI 把“规则化自我批评”引入主流对齐路径。[S045]
- 为什么重要：Claude 路线证明高质量对齐不必完全依赖人工偏好打分。[S045]
- 局限性：早期 Claude 仍主要是高质量聊天模型，agent 化与开发者工作流还没有完全展开。

### GPT-4 (2023)

- 比前代强在哪里：泛化、稳健性、考试与专业任务表现明显跃升，并进入多模态阶段。[S044]
- 核心创新：更大规模系统训练与部署安全体系。[S044]
- 为什么重要：GPT-4 抬高了闭源 frontier 的能力上限，也让“system card”成为行业重要文档形态。[S044]
- 局限性：官方未披露底层结构，外界很难从架构角度解释其能力来源。

## 4. 开源竞赛期 (2023-2024)

### Llama 2 (2023)

- 比前代强在哪里：高质量 open-weight 通用模型进入主流视野，企业和开发者可自部署。[S049]
- 核心创新：把接近前沿的 open-weight 基座做成可广泛使用的家族。[S049]
- 为什么重要：后续 Qwen、Mistral、DeepSeek、MiniMax、GLM 等开源/开放权重竞争，都建立在生态已经被打开的前提上。[S049]
- 局限性：能力仍与当时最强闭源模型有差距。

### Mixtral 8x7B (2023)

- 比前代强在哪里：在推理成本相对受控时获得更大总容量和更强表现。[S056]
- 核心创新：稀疏 MoE 在开源侧实现产品级影响力。[S056]
- 为什么重要：它之后，“MoE 是否会成为开源主流”从猜测变成现实路径。[S056]
- 局限性：MoE 提升的是容量与效率平衡，不自动解决推理、工具或产品交互问题。

### Gemini 1.5 (2024)

- 比前代强在哪里：上下文长度和原生多模态能力显著推进。[S048]
- 核心创新：1M 级 context 在主流产品中落地，多模态 family 进一步整合。[S048]
- 为什么重要：把长上下文从“演示能力”推向“产品能力”。[S048]
- 局限性：闭源模型依然缺少可审计的内部结构披露。

### Qwen2 / Qwen2.5 (2024)

- 比前代强在哪里：中文、多语言、代码、长上下文和开源可用性更均衡。[S058][S059]
- 核心创新：持续迭代的 open-weight 家族化路线，逐步把 Qwen 做成平台而不是单模型。[S058][S059]
- 为什么重要：Qwen 成为开源中文与国际多语言生态的核心参与者之一。[S058][S059]
- 局限性：这一阶段的 Qwen 仍以“高质量 dense/open family”印象更强，MoE frontier 还未成为主标签。

### DeepSeek-V3 (2024)

- 比前代强在哪里：在开放权重阵营中给出更激进的结构透明度和工程效率导向。[S015]
- 核心创新：DeepSeekMoE、MLA、MTP 的组合路线。[S015]
- 为什么重要：证明开源 frontier 并非只能模仿闭源，也可以提出自己的结构答案。[S015]
- 局限性：模型能力之外，完整 agent 化体验仍需要工具链和后训练进一步支撑。

## 5. 推理与 Agent 期 (2025-2026)

### DeepSeek-R1 (2025)

- 比前代强在哪里：推理任务表现和“先想后答”的训练范式更强。[S057]
- 核心创新：reasoning-first RL 路线被明确推到公众视野。[S057]
- 为什么重要：R1 让行业重新讨论 test-time compute、过程奖励和推理专用模型的边界。[S057]
- 局限性：强 reasoning 不自动等于强产品系统，工具、稳定性和部署体验仍是另一层问题。

### Claude 4.x / Opus 4.7 (2025-2026)

- 比前代强在哪里：agentic coding、长程任务执行、effort 与 task budget 控制更成熟。[S005][S047]
- 核心创新：把推理预算做成显式产品接口，并把模型能力和 coding agent 工作流更紧地绑在一起。[S005][S047]
- 为什么重要：前沿竞争开始从“答得像不像人”转向“能不能持续做完一项工作”。[S005]
- 局限性：底层结构继续未披露，外部只能观察接口层能力。

### Qwen3.x (2025-2026)

- 比前代强在哪里：开源家族进一步进入 MoE、混合注意力、多模态和 frontier 级推理部署竞赛。[S019][S030][S031][S060]
- 核心创新：hybrid attention 与 MoE 结合，继续做大模型家族而不只做单点型号。[S030][S031]
- 为什么重要：它展示了开源家族如何持续吸收 frontier 设计并本土化为大规模产品线。[S060]
- 局限性：产品系统能力仍不能简单从 base model config 推断。

### Kimi K2.x / K2.6 (2025-2026)

- 比前代强在哪里：从“长文本入口”升级到多模态、thinking/non-thinking、agent 任务入口。[S017][S018][S061]
- 核心创新：以长上下文和 agent 工作流为中心组织模型产品线。[S017][S061]
- 为什么重要：Kimi 的路线说明中国模型家族也在争夺“复杂工作代理”而不只是聊天体验。[S017]
- 局限性：底层训练与后训练细节不像部分开源模型那样完全透明。

### Llama 4 / Muse Spark (2025-2026)

- 比前代强在哪里：Meta 同时推进 MoE 多模态 open-weight 和产品系统模型。[S011][S033][S034]
- 核心创新：Llama 4 把 MoE、多模态和极长上下文结合；Muse Spark 则突出产品路由、感知和子代理工作流。[S011][S033][S034]
- 为什么重要：它清楚展示了“基础模型路线”和“产品系统路线”已经可以分叉演进。[S011]
- 局限性：Muse Spark 的底层结构官方仍未披露，不能和 Llama 4 混写为同一技术对象。

### Gemini 3 / GPT-5.x / GLM-5.1 / MiniMax-M2.1 / Mistral Large 3 / Grok 4.20

- 比前代强在哪里：共同把 reasoning、tool use、长上下文、稀疏化部署和 long-horizon work 往产品层推进。[S001][S009][S020][S022][S024][S025]
- 核心创新：推理预算、工具调用、产品路由、MoE 部署和低比特 serving 一起成为竞争焦点。[S001][S005][S020][S025]
- 为什么重要：到 2026 年，LLM 比拼的已不是单次问答，而是持续执行任务的可靠性、成本和系统集成能力。
- 局限性：排行榜和产品演示仍不能替代对 base model 结构、训练和评测边界的审计。
